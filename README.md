# Hackathon Ideas Factory — Specialist Swarm

**Concept landed:** Skills, plugins & sub-agents
**Tech:** [Claude Managed Agents multi-agent](https://platform.claude.com/docs/en/managed-agents/multi-agent) + [custom Skills](https://platform.claude.com/docs/en/agents-and-tools/agent-skills/overview)
**Time:** 60 minutes
**Output:** A coordinator agent that brainstorms hackathon project ideas, delegates them to 4 engineering specialist sub-agents for evaluation, and produces a complete implementation plan.

## The pitch

This is the architecture that wins hackathons and builds production-ready prototypes: **coordinator + specialists + skills**. It maps directly to how real engineering teams structure work. A lead engineer orchestrates; specialists (data, AI, platform, security) own their domains; the lead synthesizes and delivers.

You're going to build exactly that, in 60 minutes, as a Hackathon Ideas Factory. Drop a challenge brief in, get a complete project proposal with implementation plan out, watch the parallel deliberation happen in real time on the events stream.

## Setup (5 min)

You need a workspace API key on the Console (multi-agent is currently in research preview — your workspace may need to be granted access).

```bash
cd 03-specialist-swarm
pip install -r requirements.txt
export ANTHROPIC_API_KEY="sk-ant-..."
```

## Pick a scenario card

Three cards in [`scenario-cards.md`](./scenario-cards.md). Each gives you a coordinator + a different roster of specialists. Pick one. Different teams should pick different cards.

## Core build (25 min)

1. **Create the specialists.** Run `python create_specialists.py`. This creates 4 engineering sub-agents (Data Engineering, AI Engineering, Platform Engineering, Security Engineering) and saves their IDs to `.specialist_ids.json`.

2. **Upload the skills.** Run `python upload_skills.py`. This packages the custom skills in `skills/` and uploads them via the Skills API. Each specialist gets the skill that matches its domain (data-engineering-patterns, ai-engineering-playbook, platform-architecture-guide, security-checklist).

3. **Create the coordinator.** Run `python create_coordinator.py`. This creates the coordinator agent (Hackathon Lead Engineer) with `multiagent: coordinator` config, listing the specialists in its callable roster.

4. **Run the factory.** Run `python run_hackathon_factory.py`. This:
   - Loads the synthetic hackathon challenge (`synthetic-data/hackathon-challenge-climate-iot.md`)
   - Starts a session against the coordinator
   - Asks the coordinator to brainstorm 2-3 ideas and delegate to all specialists for evaluation
   - Streams the events so you can watch the parallel deliberation
   - Saves the final implementation plan to `outputs/[Idea-Name]-Proposal.md`

By minute 30 you have a complete project proposal in `outputs/`, generated by a coordinator + 4 specialists who each evaluated the ideas from their domain perspective.

## Stretch goals (20 min)

See [`stretch-goals.md`](./stretch-goals.md). The big ones:

- **Custom team-voice skill** — codify your team's engineering culture and preferences
- **Critic sub-agent** — add a fifth agent that reviews the coordinator's proposal before finalization
- **Memory across hackathons** — coordinator remembers past winning patterns and applies them
- **Synthetic MCP for tech stacks** — wire up a fake knowledge base with your team's preferred tools

## Two-minute demo

Two-monitor setup:
- **Monitor 1:** the events stream from the coordinator session, scrolling. You'll see `session.thread_created` × 4, parallel `running`, then `agent.thread_message_received` flowing back. The visible parallelism IS the demo.
- **Monitor 2:** open `outputs/[Idea-Name]-Proposal.md`. Complete implementation plan with architecture diagrams, phased breakdown, and risk assessment.

Narrate the events stream while it runs: "Watch as the Data Engineering SME, AI Engineering SME, Platform SME, and Security SME all evaluate the ideas in parallel..." The room will get it.
